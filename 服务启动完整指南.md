## 🚀 一键启动（推荐）

### 使用部署脚本

```bash
# 进入项目目录
cd /mnt/d/code/other/feilun

# 一键启动所有基础设施
./deploy-all-in-one.sh
```

**自动启动的服务**:
- ✅ Nacos (8848) - 服务发现与配置中心
- ✅ Higress (8082) - API 网关

**手动启动的服务**:
- ⚠️ Ticket 后端 (8081) - 业务 API
- ⚠️ n8n (5678) - 工作流引擎
- ⚠️ MCP Bridge (19002) - MCP 工具代理（通常已集成）

```
n8n is ready!
http://localhost:5678
```

### 步骤 3: 启动 Ticket 后端服务

#### 方式 1: Maven 运行（开发环境）

```bash
# 进入后端目录
cd /mnt/d/code/other/feilun/backend

# 使用 Maven 运行
mvn spring-boot:run

# 或者指定 Profile
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

#### 方式 2: 打包后运行

```bash
# 打包
mvn package -DskipTests

# 运行
java -jar target/ticket-management-*.jar

# 或者指定 Profile
java -jar target/ticket-management-*.jar --spring.profiles.active=dev
```

#### 方式 3: 使用 HiMarket Docker 服务（推荐）

```bash
# 进入项目根目录
cd /mnt/d/code/other/feilun

# 启动 HiMarket（包含后端、MCP Bridge、前端、数据库）
docker-compose -f docker-compose.himarket.yml up -d

# 查看日志
docker-compose -f docker-compose.himarket.yml logs -f
```

**验证**:
```bash

# http://localhost:8081/swagger-ui.html
```

**预期输出**:

```json
  "components": [

  ]
}
```

## ✅ 健康检查

### 一键健康检查脚本

```bash
#!/bin/bash
# check-all-services.sh

echo "🔍 检查所有服务状态..."

# 1. Nacos
echo -n "Nacos (8848): "
status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8848/nacos)
if [ "$status" = "200" ] || [ "$status" = "302" ]; then
  echo "✅ 正常"
else
  echo "❌ 异常 (HTTP $status)"
fi

# 2. Higress Console
echo -n "Higress Console (8001): "
status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8001)
if [ "$status" = "200" ] || [ "$status" = "302" ]; then
  echo "✅ 正常"
else
  echo "❌ 异常 (HTTP $status)"
fi

# 3. Higress Gateway
echo -n "Higress Gateway (8082): "
status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8082/)
if [ "$status" = "200" ] || [ "$status" = "404" ]; then
  echo "✅ 正常"
else
  echo "❌ 异常 (HTTP $status)"
fi

# 4. n8n
echo -n "n8n (5678): "
status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:5678/)
if [ "$status" = "200" ]; then
  echo "✅ 正常"
else
  echo "❌ 异常 (HTTP $status)"
fi

# 5. Ticket 后端
echo -n "Ticket 后端 (8081): "
status=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8081/healthz)
if [ "$status" = "200" ]; then
  echo "✅ 正常"
else
  echo "❌ 异常 (HTTP $status)"
fi

# 6. MCP Bridge
echo -n "MCP Bridge (19002): "
status=$(curl -s -o /dev/null -w "%{http_code}" http://172.28.156.225:19002/health)
if [ "$status" = "200" ]; then
  echo "✅ 正常"
else
  echo "❌ 异常 (HTTP $status)"
fi

echo ""
echo "📊 服务状态汇总"
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}" | grep -E "n8n|higress|nacos|himarket"
```

```bash
# 保存为 check-all-services.sh
chmod +x check-all-services.sh
./check-all-services.sh
```

## 🔍 常见问题

### 问题 1: 端口冲突

**症状**:
```
Error: bind: address already in use
```

**解决**:
```bash
# 查看端口占用
netstat -tunlp | grep -E "5678|8081|8082|8848"

# 或者使用 lsof
lsof -ti:5678 | xargs kill -9

# 修改端口（如果需要）
# 编辑 docker-compose.yml 或配置文件
```

### 问题 2: 容器启动失败

**症状**:
```
docker: Error response from daemon: ...
```

**解决**:
```bash
# 查看详细日志
docker logs n8n --tail 50

# 重新创建容器
docker-compose down
docker-compose up -d --force-recreate

# 清理并重启
docker system prune -a
docker-compose up -d
```

### 问题 3: 网络不通

**症状**:
```
curl: (7) Failed to connect to localhost port 5678
```

**解决**:
```bash
# 检查 Docker 网络
docker network ls
docker network inspect docker_himarket-network

# 确保所有服务在同一网络
docker network connect docker_himarket-network n8n

# 重启网络
docker-compose down
docker-compose up -d
```

### 问题 4: Nacos 启动慢

**症状**:
```
Nacos 启动超过 1 分钟
```

**解决**:
```bash
# 查看日志
docker logs nacos-standalone --tail 50

# 等待更长时间（首次启动需要初始化数据库）
for i in {1..60}; do
  curl -s http://localhost:8848/nacos && break
  echo "等待 Nacos 启动... $i/60"
  sleep 1
done
```

### 问题 5: 后端服务无法连接数据库

**症状**:
```
Error: Could not connect to MySQL
```

**解决**:
```bash
# 检查 MySQL 容器
docker ps | grep mysql

# 检查数据库连接
docker exec -it himarket-mysql mysql -uroot -p

# 检查配置文件
# backend/src/main/resources/application.yml
# 确保 MySQL 地址正确
```

## 🛑 停止服务

### 停止所有服务

```bash
# 停止所有容器（快照方式）
docker ps -q | xargs -r docker stop

# 或者分别停止
cd /mnt/d/code/other/feilun

# 停止基础设施
docker rm -f nacos-standalone higress-ai

# 停止 n8n
cd /mnt/d/code/docker/n8n && docker-compose down

# 停止 HiMarket
docker-compose -f docker-compose.himarket.yml down
```

### 停止特定服务

```bash
# 只停止 n8n
docker stop n8n

# 只停止后端
docker stop docker-himarket-server-1

# 只停止 Higress
docker stop 8fbc0236dceb_higress-ai
```

### 清理所有资源

```bash
# 停止并删除所有容器
docker ps -aq | xargs -r docker rm -f

# 清理未使用的镜像
docker image prune -a

# 清理未使用的卷
docker volume prune

# 完全清理（慎用！）
docker system prune -a --volumes
```
