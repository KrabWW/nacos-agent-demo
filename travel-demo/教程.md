# Nacos Agent 注册中心实战教程

本教程通过两个递进的项目，演示如何使用 Nacos 3.1.0+ 的 Agent 注册中心（A2A Registry）构建多 Agent 协作系统。

## 前置条件

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17+ | 本教程使用 JDK 21 |
| Nacos Server | 3.1.0+ | 支持 Agent 注册中心功能 |
| Maven | 3.8+ | 项目构建 |
| LLM API | 任意 Anthropic 兼容接口 | 本教程使用 MiniMax |

---

## 一、基础概念

### 1.1 什么是 A2A 协议？

A2A（Agent-to-Agent）是 Google 发起的开放标准，让 AI Agent 之间可以像微服务一样互相发现和调用。Nacos 从 3.1.0 开始作为 A2A 注册中心，提供 Agent 的注册、发现和版本管理。

### 1.2 核心数据模型：AgentCard

AgentCard 是描述一个 Agent 的元数据卡片，包含：

| 字段 | 说明 |
|------|------|
| `name` | Agent 唯一名称（同一命名空间下唯一） |
| `description` | Agent 能力描述 |
| `url` | A2A 协议端点地址 |
| `version` | 版本号 |
| `skills` | Agent 暴露的技能列表 |
| `provider` | 提供者信息 |

### 1.3 两种角色

```
Agent Server —— 提供 Agent，注册 AgentCard 到 Nacos
Agent Client —— 从 Nacos 发现 AgentCard，通过 A2A 协议调用远程 Agent
```

一个应用可以同时是 Server 和 Client（比如编排者 Agent）。

## 二、Demo 1：单 Agent 注册与发现

> 项目路径：`nacos-agent-demo/`

最简单的场景：一个 Agent Server 注册到 Nacos，一个 Agent Client 发现并调用它。

### 2.1 项目结构

```
nacos-agent-demo/
├── pom.xml                          # 父 POM
├── agent-server/                    # Agent 提供方（端口 9999）
│   ├── pom.xml
│   └── src/main/
│       ├── java/.../server/
│       │   ├── AgentServerApplication.java
│       │   └── RootAgentConfiguration.java    # 定义 ReactAgent
│       └── resources/application.yml
└── agent-client/                    # Agent 消费方（端口 9998）
    ├── pom.xml
    └── src/main/
        ├── java/.../client/
        │   ├── AgentClientApplication.java
        │   ├── RootAgentConfiguration.java    # 构建 A2aRemoteAgent
        │   └── AgentCallController.java       # REST 接口
        └── resources/application.yml
```

### 2.2 关键依赖

**父 POM 版本选型：**

```xml
<properties>
    <spring-ai.version>1.0.3</spring-ai.version>
    <spring-ai-alibaba.version>1.0.0.4</spring-ai-alibaba.version>
</properties>
```

**Agent Server 依赖：**

```xml
<!-- A2A Server：暴露 AgentCard 端点 -->
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-server</artifactId>
</dependency>
<!-- A2A Nacos Registry：自动注册到 Nacos -->
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-registry</artifactId>
</dependency>
<!-- LLM 客户端（本教程使用 MiniMax 的 Anthropic 兼容接口） -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-starter-model-anthropic</artifactId>
</dependency>
```

**Agent Client 依赖（不需要 LLM）：**

```xml
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-client</artifactId>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-registry</artifactId>
</dependency>
```

### 2.3 定义 Agent（Server 端）

```java
@Configuration
public class RootAgentConfiguration {

    @Bean
    @Primary
    public BaseAgent rootAgent(ChatModel chatModel) throws GraphStateException {
        return ReactAgent.builder()
                .name("nacos-helper-agent")       // Agent 名称（全局标识）
                .description("一个能回答 Nacos 问题的助手")
                .model(chatModel)                  // Spring AI 自动注入的 ChatModel
                .instruction("You are a helpful assistant...")  // System Prompt
                .build();
    }
}
```

`ReactAgent` 是 Spring AI Alibaba 提供的 ReAct 模式 Agent，它会循环执行"推理→行动→观察"直到得出答案。

### 2.4 配置 Agent 注册（Server 端）

```yaml
spring:
  ai:
    anthropic:                              # LLM 配置
      api-key: ${MINIMAX_API_KEY}
      base-url: https://api.minimaxi.com/anthropic
      chat.options.model: MiniMax-M1
    alibaba:
      a2a:
        nacos:
          server-addr: localhost:8848       # Nacos 地址
          username: nacos
          password: nacos
          registry:
            enabled: true                   # 必须设为 true！开启自动注册
        server:
          version: 1.0.0                    # AgentCard 版本
          card:
            url: http://localhost:9999/a2a  # A2A 端点地址
```

> **踩坑记录：** `spring.ai.alibaba.a2a.nacos.registry.enabled=true` 是必须的，不加这个配置 Agent 不会注册到 Nacos。

### 2.5 发现并调用 Agent（Client 端）

```java
@Configuration
public class RootAgentConfiguration {

    @Bean
    public BaseAgent remoteAgent(AgentCardProvider agentCardProvider)
            throws GraphStateException {
        return A2aRemoteAgent.builder()
                .agentCardProvider(agentCardProvider)  // 自动从 Nacos 发现
                .name("nacos-helper-agent")             // 要发现的 Agent 名称
                .build();
    }
}
```

```yaml
spring.ai.alibaba.a2a.nacos:
  discovery.enabled: true         # 开启自动发现
  server-addr: localhost:8848
```

`AgentCardProvider` 由框架自动注入，它会根据 `name` 从 Nacos 订阅 AgentCard 和端点信息。Agent 发布新版本或端点变化时自动推送。

### 2.6 启动与验证

```bash
# 1. 启动 Nacos
docker compose -f nacos-docker-compose.yml up -d

# 2. 启动 Agent Server
cd agent-server && mvn spring-boot:run

# 3. 验证注册成功
TOKEN=$(curl -s -X POST 'localhost:8848/nacos/v1/auth/login' \
    -d 'username=nacos&password=nacos' | python3 -c "import sys,json;print(json.load(sys.stdin)['accessToken'])")

curl "localhost:8848/nacos/v3/admin/ai/a2a?namespaceId=public&agentName=nacos-helper-agent&accessToken=$TOKEN"

# 4. 启动 Agent Client
cd agent-client && mvn spring-boot:run

# 5. 调用测试
curl "http://localhost:9998/ask?question=What%20is%20Nacos?"
```

## 三、Demo 2：多 Agent 协作——旅行规划系统

> 项目路径：`travel-demo/`

真实场景中，一个用户请求往往需要多个 Agent 协作完成。本 Demo 模拟旅行规划：编排者 Agent 同时调用天气、酒店、机票三个专业 Agent。

### 3.1 架构图

```
用户 ──HTTP──► Travel Planner（编排者）:10010
                    │
                    │  从 Nacos 发现并并行调用 3 个远程 Agent
                    ├──► Weather Agent :10011  → "上海下周天气如何？"
                    ├──► Hotel Agent   :10012  → "推荐上海酒店"
                    └──► Flight Agent  :10013  → "北京到上海的航班"
                    │
                    │  聚合结果返回给用户
                    ▼
              { weather: "...", hotel: "...", flight: "..." }
```

### 3.2 项目结构

```
travel-demo/
├── pom.xml                  # 父 POM
├── weather-agent/           # 天气 Agent（Server only）
├── hotel-agent/             # 酒店 Agent（Server only）
├── flight-agent/            # 机票 Agent（Server only）
└── travel-planner/          # 编排者（Server + Client）
```

### 3.3 三个专业 Agent（Server only）

每个 Worker Agent 结构完全相同，只是端口、名称和 System Prompt 不同：

| Agent | 端口 | 名称 | 职责 |
|-------|------|------|------|
| Weather | 10011 | travel-weather-agent | 查询城市天气，提供出行建议 |
| Hotel | 10012 | travel-hotel-agent | 推荐酒店，提供住宿建议 |
| Flight | 10013 | travel-flight-agent | 查询航班，提供机票建议 |

以 Weather Agent 为例：

```java
@Bean
@Primary
public BaseAgent weatherAgent(ChatModel chatModel) throws GraphStateException {
    return ReactAgent.builder()
            .name("travel-weather-agent")
            .description("查询城市天气信息，提供出行天气建议")
            .model(chatModel)
            .instruction("你是一个天气查询助手。根据用户提供的城市和日期，"
                    + "给出天气情况的建议。用中文回答，简洁明了。")
            .build();
}
```

```yaml
server.port: 10011
spring.ai.alibaba.a2a:
  nacos:
    server-addr: localhost:8848
    registry.enabled: true          # 注册到 Nacos
  server:
    version: 1.0.0
    card.url: http://localhost:10011/a2a
```

### 3.4 编排者 Agent（Server + Client）

Travel Planner 是核心——它既是一个 Agent Server（对用户暴露），又是 Agent Client（调用其他 Agent）。

**依赖：同时引入 server + client + registry**

```xml
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-server</artifactId>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-client</artifactId>
</dependency>
<dependency>
    <groupId>com.alibaba.cloud.ai</groupId>
    <artifactId>spring-ai-alibaba-starter-a2a-registry</artifactId>
</dependency>
```

**配置：同时开启 registry 和 discovery**

```yaml
spring.ai.alibaba.a2a.nacos:
  registry.enabled: true        # 作为 Server 注册自己
  discovery.enabled: true       # 作为 Client 发现其他 Agent
```

**构建 3 个远程 Agent Bean：**

```java
@Configuration
public class RemoteAgentsConfig {

    @Bean
    public BaseAgent weatherAgent(AgentCardProvider provider) throws GraphStateException {
        return A2aRemoteAgent.builder()
                .agentCardProvider(provider)
                .name("travel-weather-agent")
                .build();
    }

    @Bean
    public BaseAgent hotelAgent(AgentCardProvider provider) throws GraphStateException {
        return A2aRemoteAgent.builder()
                .agentCardProvider(provider)
                .name("travel-hotel-agent")
                .build();
    }

    @Bean
    public BaseAgent flightAgent(AgentCardProvider provider) throws GraphStateException {
        return A2aRemoteAgent.builder()
                .agentCardProvider(provider)
                .name("travel-flight-agent")
                .build();
    }
}
```

**编排逻辑——并行调用：**

```java
@GetMapping("/plan")
public Map<String, String> plan(@RequestParam("question") String question) {
    // 并行调用三个远程 Agent
    CompletableFuture<String> weatherFuture = CompletableFuture.supplyAsync(
            () -> callAgent(weatherAgent, "用户正在规划旅行：" + question + "，请提供天气建议"));
    CompletableFuture<String> hotelFuture = CompletableFuture.supplyAsync(
            () -> callAgent(hotelAgent, "用户正在规划旅行：" + question + "，请推荐酒店"));
    CompletableFuture<String> flightFuture = CompletableFuture.supplyAsync(
            () -> callAgent(flightAgent, "用户正在规划旅行：" + question + "，请推荐航班"));

    // 等待所有结果
    CompletableFuture.allOf(weatherFuture, hotelFuture, flightFuture).join();

    return Map.of("question", question,
                  "weather", weatherFuture.join(),
                  "hotel", hotelFuture.join(),
                  "flight", flightFuture.join());
}
```

## 四、关键知识点总结

### 4.1 自动配置条件链

```
BaseAgent Bean 存在
  → A2aServerAgentCardAutoConfiguration 创建 AgentCard
    → AgentRegistry Bean 存在（需要 nacos.registry.enabled=true）
      → A2aServerRegistryAutoConfiguration 注册到 Nacos
```

### 4.2 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| Nacos 里看不到 Agent | 缺少 `registry.enabled: true` | 加上该配置 |
| Agent 启动报错找不到 ChatModel | 缺少 LLM 客户端依赖 | 加入 `spring-ai-starter-model-anthropic` |
| Client 调用超时 | Agent Server 未启动或端口不通 | 先启动 Server，检查 `card.url` 配置 |
| Nacos API 返回 access denied | 鉴权未通过 | 先 `/v1/auth/login` 获取 token |
| `list` 接口返回空但 Agent 存在 | `search=blur` 需要带 `agentName` 参数 | 用精确查询或带空 agentName |

### 4.3 从单 Agent 到多 Agent 的演进思路

```
单 Agent：用户 → Agent Server → LLM
         Agent Server 自己就是最终执行者

多 Agent：用户 → 编排者 Agent → [天气 Agent, 酒店 Agent, 机票 Agent]
         编排者不直接执行业务，而是分发到专业 Agent
         每个专业 Agent 独立部署、独立演进
```

这就是"像管理微服务一样管理 Agent"的核心思想。

## 五、扩展方向

1. **动态发现**：新增 Agent 不需要改编排者代码，只要注册到 Nacos 即可
2. **负载均衡**：同一 Agent 部署多个实例，Nacos 自动感知
3. **版本管理**：AgentCard 支持多版本，可灰度发布
4. **按技能检索**：未来 Nacos 支持按 skills/tags 过滤 Agent
5. **A2A 标准协议**：待官方 Registry Protocol 发布后，Nacos 会适配标准协议
