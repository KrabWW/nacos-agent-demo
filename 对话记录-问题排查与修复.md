### 步骤 5: 查看错误详情

```bash
n8n_executions(action: "get", id: "38", mode: "error")

# 发现真正的问题:
```

**❌ 错误 1: JSON 表达式格式错误**
```json
{
  "message": "The value in the \"JSON Body\" field is not valid JSON",
  "nodeName": "创建订单 (MCP)",
  "nodeType": "n8n-nodes-base.httpRequest"
}
```

**配置错误**:
```javascript
// ❌ 错误的配置
"jsonBody": "= {\n    \"product_id\": {{ $json.productId }},\n    \"quantity\": {{ $json.quantity }},\n    \"address\": \"默认地址\"\n  }"

// 问题:
// 1. 使用了错误的语法 `= {{ }}`
// 2. 引用了不存在的字段 `$json.productId`
// 3. 实际数据结构是 `$json.result.items[0].id`
```

**❌ 错误 2: SSRF 保护阻止**
```
SSRF protection: Localhost access is blocked in strict mode
原因: n8n 默认阻止访问内网 IP (172.28.156.225)
```

---

## 🛠️ 修复方案

### 修复 1: 更新工作流节点配置

```bash
# 使用 n8n MCP 工具更新节点
n8n_update_partial_workflow(
  id: "eOPjpVDChTp8Eft5",
  operations: [{
    type: "updateNode",
    nodeName: "创建订单 (MCP)",
    updates: {
      parameters: {
        method: "POST",
        url: "http://172.28.156.225:19002/tools/create_order",
        jsonBody: "={{ JSON.stringify({\n  \"product_id\": $json.result.items[0].id,\n  \"quantity\": 1,\n  \"address\": \"默认地址\"\n}) }}"
      }
    }
  }]
)

# 结果: ✅ 工作流更新成功
```

**修复要点**:
1. 使用 `JSON.stringify()` 确保输出有效 JSON
2. 正确引用上游数据: `$json.result.items[0].id`
3. 使用表达式前缀 `={{ }}`

### 修复 2: 配置环境变量

```yaml
# 文件: /mnt/d/code/docker/n8n/docker-compose.yml
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    environment:
      # 允许访问内网地址（开发环境）
      - N8N_BLOCK_IP_HEADLESS_REQUEST=
      - N8N_BLOCK_IP_HOST_REQUEST_FOR_ENV=
      - N8N_NODE_EXECUTION_ALLOW_REGEX=.*
```

```bash
# 重启 n8n 容器
cd /mnt/d/code/docker/n8n
docker-compose down
docker-compose up -d
```

## 💡 经验教训

### 1. 数据流追踪的重要性

**错误**: 直接假设字段名 `$json.productId`
**正确**: 查看上游节点的实际输出 `$json.result.items[0].id`

**方法**:
```bash
# 查看执行详情，了解每个节点的实际数据
n8n_executions(action: "get", id: "execution_id", mode: "error")
```

### 2. n8n 表达式语法

**正确格式**:
```javascript
// 方法 1: JSON.stringify（推荐）
={{ JSON.stringify({"key": value})}}

// 方法 2: 直接对象
={{ {"key": value} }}

// 方法 3: 字符串拼接
={{ "文本" + $json.field }}
```

### 3. SSRF 保护配置

**开发环境**:
```yaml
environment:
  - N8N_BLOCK_IP_HEADLESS_REQUEST=
```

**生产环境**:
```yaml
environment:
  - N8N_ALLOWED_HOSTS=172.28.156.225,api.internal.com
```

### 4. 工具优先级

**推荐工具使用顺序**:
1. n8n MCP 工具（快速诊断）
2. n8n Web UI（查看详细配置）
3. 执行历史（错误详情）
4. Docker 日志（底层问题）
