## 🔴 问题总结

### 问题 1: JSON 表达式格式错误 ✅ 已修复

**问题描述**:
- Webhook 调用后工作流执行失败
- 无法正确创建订单

**错误信息**:
```
NodeOperationError: The value in the "JSON Body" field is not valid JSON
```

**根本原因**:
1. 工作流中 "创建订单 (MCP)" 节点的 JSON Body 配置错误
2. 使用了错误的 n8n 表达式语法
3. 引用了不存在的字段路径 `$json.productId`（实际应为 `$json.result.items[0].id`）

**修复前**:
```javascript
// ❌ 错误的配置
"jsonBody": "= {\n    \"product_id\": {{ $json.productId }},\n    \"quantity\": {{ $json.quantity }},\n    \"address\": \"默认地址\"\n  }"
```

**修复后**:
```javascript
// ✅ 正确的配置
"jsonBody": "={{ JSON.stringify({\n  \"product_id\": $json.result.items[0].id,\n  \"quantity\": 1,\n  \"address\": \"默认地址\"\n}) }}"
```

**修复方式**:
- 使用 n8n MCP 工具更新工作流节点配置
- 使用 `JSON.stringify()` 确保输出有效的 JSON
- 正确引用上游数据结构

### 问题 2: SSRF 保护阻止 ✅ 已修复

**问题描述**:
- 工作流无法访问内网 MCP 服务
- 所有 HTTP Request 节点执行失败

**错误信息**:
```
SSRF protection: Localhost access is blocked in strict mode
```

**根本原因**:
1. n8n 默认启用 SSRF (Server-Side Request Forgery) 保护
2. 阻止对内网 IP 地址的访问
3. HTTP Request 节点尝试访问 `http://172.28.156.225:19002`

**修复方案**:
```yaml
# docker-compose.yml
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      # 允许访问内网地址（开发环境）
      - N8N_BLOCK_IP_HEADLESS_REQUEST=
      # 允许访问本地和内网 IP
      - N8N_BLOCK_IP_HOST_REQUEST_FOR_ENV=
      # 开发环境：禁用严格的安全检查
      - N8N_NODE_EXECUTION_ALLOW_REGEX=.*
    volumes:
      - n8n_data:/home/node/.n8n
```

**注意事项**:
⚠️ 这些配置仅适用于开发环境
⚠️ 生产环境应配置具体的 IP 白名单，而非完全禁用保护

节点: Webhook

  "productName": "iPhone 16",

  "success": true,

  "result": {
    "total": 2,

        "id": 1,
        "name": "iPhone 16",

节点: 提取商品信息

### 2. JSON 表达式语法

#### ✅ 正确的表达式格式

```javascript
// 方法 1: JSON.stringify（推荐）
={{
  JSON.stringify({
    "product_id": $json.result.items[0].id,
    "quantity": 1,
    "address": "默认地址"
  })
}}

// 方法 2: 直接对象（简单场景）
={{
  "product_id": $json.result.items[0].id,
  "quantity": 1
}}

// 方法 3: 字符串拼接
={{ "商品ID: " + $json.id + ", 数量: " + $json.quantity }}
```

#### ❌ 错误的表达式格式

```javascript
// 错误 1: 使用 = 前缀
"= {\n  \"product_id\": {{ $json.id }}\n}"

// 错误 2: 引用不存在的字段
{{ $json.productId }}  // 应该是 $json.result.items[0].id

// 错误 3: 混用表达式和 JSON
"= {{ "product_id": $json.id }}"
```

### 3. 数据引用规则

```javascript
// 当前节点数据
$json

// 上一个节点的数据
$json.fieldName

// 嵌套字段
$json.result.items[0].id

// 数组访问
$json.items[0]

// 条件表达式
$json.status === 'success' ? '成功' : '失败'

// 默认值
$json.quantity || 1

// 安全访问
$json['nested.field']
```

## 🔍 故障排查

### 常见问题与解决方案

#### 问题 1: Webhook 无响应

**症状**:
```bash
curl -X POST "http://localhost:5678/webhook/product-search-demo" \
  -d '{"productName": "test"}'
# 无返回，一直等待
```

**排查步骤**:
1. 检查工作流是否激活
```bash
curl -H "X-N8N-API-KEY: $API_KEY" \
  http://localhost:5678/api/v1/workflows/eOPjpVDChTp8Eft5 | jq '.active'
```

2. 检查是否有 Respond to Webhook 节点
3. 查看 n8n 日志
```bash
docker logs n8n --tail 50
```

4. 检查执行历史
```bash
curl -H "X-N8N-API-KEY: $API_KEY" \
  "http://localhost:5678/api/v1/executions?workflowId=eOPjpVDChTp8Eft5"
```

#### 问题 2: JSON 解析错误

**症状**:
```
NodeOperationError: The value in the "JSON Body" field is not valid JSON
```

**解决方案**:
1. 使用 `JSON.stringify()` 包裹对象
2. 检查字段路径是否正确
3. 验证上游数据结构

#### 问题 3: SSRF 保护阻止

**症状**:
```
SSRF protection: Localhost access is blocked in strict mode
```

**解决方案**:
1. 添加环境变量到 docker-compose.yml
2. 重启容器
3. 检查目标 IP 是否在允许列表中

#### 问题 4: 超时错误

**症状**:
```
Error: Request timed out after 30000ms
```

**解决方案**:
1. 在 HTTP Request 节点中增加超时时间
2. 检查目标服务是否正常
3. 查看网络连接

## 🔍 MCP Bridge 价值分析

### 为什么需要 MCP Bridge？

#### 直观对比：直接调用 API vs 使用 MCP Bridge

| 维度 | 直接调用后端 API | 通过 MCP Bridge |
|------|----------------|-----------------|
| **AI 理解成本** | ❌ 需要阅读 Swagger 文档 | ✅ 结构化工具描述 |
| **参数格式** | ❌ 硬编码字段映射 | ✅ 自动参数解析 |
| **错误处理** | ❌ 手动处理 HTTP 错误 | ✅ 统一错误响应 |
| **能力发现** | ❌ 无法动态发现 | ✅ 自动注册到 Nacos |
| **协议兼容** | ❌ 仅 REST API | ✅ 支持 SSE/MCP 协议 |
| **AI 集成** | ❌ 需要 Prompt 工程技巧 | ✅ AI 原生理解 |

---

### MCP Bridge 的核心价值

#### 1. **AI 原生能力描述**

```json
// ❌ 直接调用 API - AI 不知道这是什么
curl http://localhost:8081/api/products/search?keyword=iPhone

// ✅ MCP Bridge - AI 能理解工具能力
{
  "name": "search_products",
  "description": "搜索商品，支持关键词匹配",
  "inputSchema": {
    "type": "object",
    "properties": {
      "keyword": {
        "type": "string",
        "description": "搜索关键词，如：iPhone, MacBook"
      }
    },
    "required": ["keyword"]
  }
}
```

**价值**: 
- AI 不需要阅读代码就能理解工具用途
- 自动生成正确的调用代码
- 减少幻觉和错误调用

---

#### 2. **统一协议与抽象层**

```
┌─────────────────────────────────────────────────────────┐
│                 AI Agent / n8n 工作流                    │
└──────────────────┬──────────────────────────────────────┘
                   ↓
┌──────────────────────────────────────────────────────────┐
│              MCP Bridge (协议转换层)                      │
│  • REST API → MCP Protocol                              │
│  • 参数验证与格式化                                      │
│  • 错误处理与重试                                        │
│  • 能力描述与元数据                                      │
└──────────────────┬───────────────────────────────────────┘
                   ↓
        ┌──────────┴──────────┐
        ↓                     ↓
┌──────────────┐      ┌──────────────┐
│ Ticket API   │      │ 邮件 API     │
│ (商品/订单)   │      │ (通知服务)    │
└──────────────┘      └──────────────┘
```

**价值**:
- **解耦合**: AI 不需要关心底层 API 细节
- **可替换**: 后端 API 变更不影响 AI 调用
- **可扩展**: 轻松添加新的后端服务

---

#### 3. **能力自动注册与发现**

```bash
# MCP Bridge 自动注册能力到 Nacos
POST http://localhost:8848/nacos/v1/mcp/tools
{
  "name": "search_products",
  "endpoint": "http://172.28.156.225:19002/tools/search_products",
  "category": "ecommerce"
}

# AI Agent 可以动态发现能力
GET http://localhost:8848/nacos/v1/mcp/tools
```

**价值**:
- **能力目录**: 所有工具集中管理
- **动态发现**: 新服务自动接入
- **版本管理**: 支持能力版本演进

---

#### 4. **安全与访问控制**

```yaml
# MCP Bridge 配置示例
mcpBridge:
  authService:
    - name: "search_products"
      allowedRoles: ["user", "admin"]
      rateLimit: 100/min
    
    - name: "delete_order"
      allowedRoles: ["admin"]
      rateLimit: 10/min
```

**价值**:
- **权限控制**: 细粒度的访问权限
- **限流保护**: 防止滥用和攻击
- **审计日志**: 记录所有工具调用

---

#### 5. **增强的错误处理**

```javascript
// ❌ 直接调用 API - 错误处理复杂
try {
  const response = await fetch('http://api.example.com/products');
  if (response.status === 500) {
    // 服务器错误
  } else if (response.status === 404) {
    // 未找到
  }
} catch (error) {
  // 网络错误
}

// ✅ MCP Bridge - 统一错误响应
{
  "success": false,
  "error": {
    "code": "PRODUCT_NOT_FOUND",
    "message": "商品不存在",
    "retryable": false
  }
}
```

**价值**:
- **统一格式**: 错误消息结构化
- **自动重试**: 可配置的重试策略
- **降级处理**: 服务降级时返回默认值
